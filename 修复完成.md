# 权限显示"未知"问题 - 修复完成 ✅

## 问题
超级管理员分配案件权限后显示"未知"，无法修改权限。

## 根本原因
1. 后端返回 `role` 字段，前端期望 `permission_level` 字段
2. 数据库存储 `user` 值，前端没有对应的映射

## 修复方案

### 后端修改
**文件**: `backend/api/v1/case_permissions.py`
- ✅ 兼容 `role` 和 `permission_level` 两种字段名
- ✅ 返回数据同时包含两个字段
- ✅ 将 `read`/`write` 映射为数据库的 `user`

### 前端修改
**文件**: 
- `frontend/src/views/System/CaseManagement/index.vue`
- `frontend/src/views/System/CaseManagement/PermissionDialog.vue`

**修改内容**:
```javascript
function getPermissionLabel(permission) {
  const labelMap = {
    admin: '管理员',
    write: '读写',
    read: '只读',
    user: '读写'  // 将后端的 user 映射为"读写"
  }
  return labelMap[permission] || '未知'
}
```

## 测试步骤

1. **重启后端服务**
   ```bash
   cd backend
   python main.py
   ```

2. **刷新前端页面**

3. **测试功能**
   - 进入案件管理 → 权限管理
   - 添加权限：选择用户和权限级别
   - 查看权限列表：应显示"管理员"或"读写"
   - 修改权限：可以正常修改
   - 案件列表：查看"我的权限"列

## 预期结果
- ✅ 权限显示为：管理员、读写、只读
- ✅ 不再显示"未知"
- ✅ 可以正常添加、修改、撤销权限

## 注意事项
当前数据库只有 `admin` 和 `user` 两种权限：
- 前端选择"读写"或"只读"时，后端都存储为 `user`
- 显示时统一显示为"读写"
- 如需真正区分三级权限，需修改数据库表结构（详见 BUGFIX_权限显示未知.md）
