# 案件表结构优化完成

## 实现功能

根据用户需求，在生成新案件自动生成表结构时，实现了以下两个功能：

### 1. ID从1开始

所有表的 `AUTO_INCREMENT` 值已设置为从 1 开始（MySQL不支持从0开始）。

**修改前：**
```sql
) ENGINE=InnoDB AUTO_INCREMENT=159695 DEFAULT CHARSET=utf8mb4;
```

**修改后：**
```sql
) ENGINE=InnoDB AUTO_INCREMENT=1 DEFAULT CHARSET=utf8mb4;
```

### 2. case_no字段默认值为案件编号

对于所有包含 `case_no` 字段的表（共19个表），将其默认值设置为案件编号占位符 `{{CASE_CODE}}`，在创建案件数据库时会自动替换为实际的案件编号。

**修改前：**
```sql
`case_no` varchar(255) DEFAULT '3003xpj' COMMENT '案件编号',
-- 或
`case_no` varchar(255) DEFAULT NULL COMMENT '案件编号',
```

**修改后：**
```sql
`case_no` varchar(255) DEFAULT '{{CASE_CODE}}' COMMENT '案件编号',
```

## 使用方法

创建新案件时，系统会自动：

1. 创建独立的案件数据库
2. 初始化76个表结构
3. 所有表的ID从1开始
4. 所有包含 `case_no` 字段的表，其默认值自动设置为案件编号

无需手动操作，完全自动化。
