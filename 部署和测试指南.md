### 2. 测试上传接口

#### 使用curl测试
```bash
# 1. 先登录获取token
curl -X POST "http://localhost:8000/api/v1/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"username": "your_username", "password": "your_password"}'

# 2. 上传文件（替换{case_id}和{token}）
curl -X POST "http://localhost:8000/api/v1/cases/{case_id}/bank-statements/upload" \
  -H "Authorization: Bearer {token}" \
  -F "files=@/path/to/your/file.zip"
```

#### 使用Python测试
```python
import requests

# 登录
login_response = requests.post(
    "http://localhost:8000/api/v1/auth/login",
    json={"username": "your_username", "password": "your_password"}
)
token = login_response.json()["data"]["access_token"]

# 上传文件
case_id = 1  # 替换为实际的案件ID
files = {"files": open("/path/to/your/file.zip", "rb")}
headers = {"Authorization": f"Bearer {token}"}

upload_response = requests.post(
    f"http://localhost:8000/api/v1/cases/{case_id}/bank-statements/upload",
    files=files,
    headers=headers
)
print(upload_response.json())

# 获取task_id
task_id = upload_response.json()["data"]["task_id"]

# 查询进度
progress_response = requests.get(
    f"http://localhost:8000/api/v1/cases/{case_id}/bank-statements/tasks/{task_id}",
    headers=headers
)
print(progress_response.json())
```

## 六、常见问题

### 1. Redis连接失败
```
错误: Error connecting to Redis
解决: 确保Redis服务正在运行
```

### 2. Celery任务不执行
```
问题: 任务状态一直是pending
解决: 检查Celery worker是否正在运行
```

### 3. 文件上传失败
```
问题: 413 Request Entity Too Large
解决: 增加nginx或FastAPI的文件大小限制
```

### 4. 数据库连接失败
```
问题: Can't connect to MySQL server
解决: 检查.env文件中的数据库配置
```

### 5. 权限不足
```
问题: 403 权限不足
解决: 确保用户对案件有相应权限（write/admin）
```

## 七、性能优化建议

1. **调整Celery并发数**
```bash
celery -A backend.core.celery_app worker --concurrency=4 --loglevel=info
```

2. **调整线程池大小**
修改 `bank_statement_service.py` 中的线程数（默认25000可能过大）

3. **使用Redis持久化**
```bash
# 在redis.conf中启用
appendonly yes
```

4. **定期清理已处理文件**
```bash
# 创建定时任务清理30天前的文件
find ./data/bank_statements/processed/ -mtime +30 -delete
```

## 八、生产环境部署

### 1. 使用Supervisor管理Celery
```ini
[program:celery_worker]
command=/path/to/venv/bin/celery -A backend.core.celery_app worker --loglevel=info
directory=/Users/yipf/DataPivot项目/DataPivot
user=your_user
autostart=true
autorestart=true
```

### 2. 使用Nginx反向代理
```nginx
location /api/ {
    proxy_pass http://127.0.0.1:8000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    client_max_body_size 100M;  # 允许大文件上传
}
```

### 3. 配置环境变量
在 `.env` 文件中添加：
```
REDIS_HOST=localhost
REDIS_PORT=6379
```
