## 使用方式

### 通过 API 创建案件

现在通过 API 创建案件时，会自动完成以下操作：

```bash
POST /api/v1/cases
{
  "case_name": "新案件",
  "case_code": "CASE001",
  "description": "案件描述"
}
```

**自动执行**:
1. ✓ 创建案件记录（datapivot.cases 表）
2. ✓ 创建案件数据库（如：新案件_CASE001）
3. ✓ 初始化 76 个表结构（从 fx_test 模板）
4. ✓ 分配创建者 admin 权限

### 通过代码创建案件

```python
from backend.services.case_service import create_case_database

# 创建案件数据库（自动初始化表结构）
database_name = create_case_database(
    case_id=1,
    case_name="测试案件",
    case_code="TEST01"
)

# 返回: "测试案件_TEST01"
# 数据库已创建，76个表已初始化
```

## 技术细节

### 表结构来源
- 基于 `fx_test` 数据库结构
- 模板文件: `sql/schema/case_template.sql` (134KB)
- 包含 76 个表的完整定义

### 错误处理
- 如果模板文件不存在，抛出异常
- 如果表结构初始化失败，自动删除已创建的数据库
- 确保数据一致性，避免创建不完整的数据库

### 性能
- 初始化 76 个表约需 2-3 秒
- 使用事务确保原子性
- 失败时自动回滚

## 优势

### 1. 自动化
- 无需手动创建表结构
- 无需手动执行 SQL 脚本
- 一键创建完整案件数据库

### 2. 一致性
- 所有案件数据库结构完全一致
- 基于统一的 fx_test 模板
- 避免人为错误

### 3. 可维护性
- 表结构更新只需重新导出模板
- 运行 `python export_fx_test_schema.py`
- 新案件自动使用最新结构

### 4. 安全性
- 失败自动回滚
- 不会留下不完整的数据库
- 事务保证数据一致性

## 维护说明

### 更新表结构模板

如果 fx_test 数据库结构有更新：

```bash

python export_fx_test_schema.py

# 2. 新创建的案件会自动使用新模板
# 3. 已存在的案件数据库不受影响
```
