## 📋 API端点列表

| 方法 | 端点 | 功能 | 权限 |
|------|------|------|------|
| POST | `/api/v1/cases/{case_id}/bank-statements/upload` | 上传文件 | write |
| GET | `/api/v1/cases/{case_id}/bank-statements/tasks/{task_id}` | 查询进度 | read |
| GET | `/api/v1/cases/{case_id}/bank-statements/tasks` | 任务列表 | read |
| POST | `/api/v1/cases/{case_id}/bank-statements/tasks/{task_id}/cancel` | 取消任务 | write |
| DELETE | `/api/v1/cases/{case_id}/bank-statements/tasks/{task_id}` | 删除任务 | admin |
| GET | `/api/v1/cases/{case_id}/bank-statements/statistics` | 获取统计 | read |

## 🚀 启动步骤

### 1. 执行数据库迁移
```bash
mysql -u root -p datapivot < backend/migrations/add_bank_statement_tasks.sql
```

### 2. 启动Redis
```bash
# macOS使用Homebrew
brew services start redis

# 或手动启动
redis-server

# 验证
redis-cli ping  # 应返回 PONG
```

### 3. 启动Celery Worker（新终端窗口）
```bash
cd /Users/yipf/DataPivot项目/DataPivot
source venv/bin/activate
celery -A backend.core.celery_app worker --loglevel=info
```

### 4. 启动FastAPI应用（新终端窗口）
```bash
cd /Users/yipf/DataPivot项目/DataPivot/backend
source ../venv/bin/activate
python main.py
```

### 5. 访问API文档
打开浏览器访问：http://localhost:8000/docs

## 🎯 核心特性

1. **案件隔离**：每个案件使用独立的数据库
2. **异步处理**：避免HTTP超时，支持长时间运行
3. **进度跟踪**：实时查询处理进度和状态
4. **权限控制**：集成现有的read/write/admin权限
5. **错误处理**：文件级错误隔离，详细错误日志
6. **文件管理**：自动归档和清理机制
7. **数据复用**：完全复用现有脚本的数据处理逻辑

## 🔧 技术栈

- **Web框架**：FastAPI
- **异步任务**：Celery + Redis
- **数据处理**：Pandas + NumPy
- **日志**：Loguru
- **数据库**：MySQL（动态切换案件数据库）
- **文件上传**：python-multipart

## 📝 使用流程

1. 用户登录系统
2. 选择或创建案件
3. 上传银行流水压缩包（支持多文件）
4. 系统返回task_id
5. 后台异步处理：解压→清洗→导入
6. 实时查询处理进度
7. 处理完成后查看统计信息
8. 数据已导入到案件数据库
