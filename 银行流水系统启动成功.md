## 已修复的问题

### 1. 配置文件问题
**问题**: Pydantic Settings 无法读取 `.env` 文件
**原因**:
- `.env` 文件使用了带空格的赋值语法 (`KEY = value`)
- 从 `backend/` 目录运行时路径不正确
- Pydantic 将环境变量转为小写，但字段定义是大写

**解决方案**:
- 修改 `.env` 文件格式为 `KEY=value`（无空格）
- 在 `config.py` 中添加 `get_project_root()` 函数动态定位 `.env` 文件
- 配置 `case_sensitive=False` 使 Pydantic 不区分大小写

### 2. Celery 任务注册问题
**问题**: Celery Worker 启动后没有发现任务
**原因**: `autodiscover_tasks()` 配置不正确

**解决方案**:
- 在 Celery 初始化时直接使用 `include=['backend.tasks.bank_statement_tasks']`
- 移除 `autodiscover_tasks()` 调用

### 3. Shell 脚本换行符问题
**问题**: Shell 脚本无法执行，提示 `bad interpreter: /bin/bash^M`
**原因**: 脚本包含 Windows 风格的换行符 (CRLF)

**解决方案**:
- 使用 `sed -i '' 's/\r$//' scripts/*.sh` 转换为 Unix 换行符 (LF)
