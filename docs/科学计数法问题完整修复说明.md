# 科学计数法问题完整修复说明

## 问题发现过程

### 第一次发现（2026-03-09）
用户报告：生成的测试数据文件中卡号变成科学计数法
```
6217001234567890123 → 6.21700123456789e+18
```

**初步修复**：使用 openpyxl 生成文件，设置文本格式

### 第二次发现（2026-03-09）
用户再次报告：导入测试文件时依然显示科学计数法
```
第2行 - 卡号: 6.21700123456789e+18 - 无效的卡类型: c15q
```

**根本原因**：虽然 Excel 文件中是文本格式，但 pandas 读取时会自动转换为数字

## 完整解决方案

### 修复点1：生成 Excel 文件（测试数据）

**文件**：`generate_test_data.py`

**方法**：使用 openpyxl 设置单元格为文本格式

```python
from openpyxl import Workbook

wb = Workbook()
ws = wb.active

# 添加数据
for row_data in test_data:
    ws.append(row_data)

# 设置卡号列为文本格式
for row in range(2, ws.max_row + 1):
    cell = ws.cell(row=row, column=1)
    cell.number_format = '@'  # @ 表示文本格式

wb.save(output_path)
```

### 修复点2：读取 Excel 文件（导入服务）⭐

**文件**：`backend/services/case_card_service.py`

## 技术原理

### pandas 的自动类型推断

```python
# pandas 读取 Excel 时的行为
df = pd.read_excel(file)

# 对于数字列，pandas 会：
# 1. 尝试转换为数值类型（int64 或 float64）
# 2. 如果数字超过15位，使用 float64 存储
# 3. float64 精度有限，导致精度丢失
# 4. 打印时显示为科学计数法
```

### dtype 参数的作用

```python
# 使用 dtype 参数
df = pd.read_excel(file, dtype={'卡号': str})

# pandas 会：
# 1. 跳过类型推断
# 2. 直接将单元格内容读取为字符串
# 3. 保持原始值不变
# 4. 避免精度丢失
```

## 其他注意事项

### 1. 导入模板生成

如果系统提供导入模板下载功能，也需要设置卡号列为文本格式：

```python
# 在模板生成代码中
from openpyxl import Workbook

wb = Workbook()
ws = wb.active
ws.append(['卡号', '卡类型', '卡主姓名', '用户ID', '批次'])

# 设置卡号列为文本格式
for row in range(2, 100):  # 预留100行
    cell = ws.cell(row=row, column=1)
    cell.number_format = '@'

wb.save('template.xlsx')
```

### 2. 用户手动填写

在模板中添加说明：
```
注意：卡号列必须设置为文本格式
方法1：在卡号前加单引号 '6217001234567890123
方法2：先设置单元格格式为文本，再输入卡号
```

### 3. 其他长数字字段

如果系统中还有其他长数字字段（如身份证号、手机号），也需要：
- 生成时设置为文本格式
- 读取时使用 dtype 参数

## 经验总结

### 问题根源
pandas 的自动类型推断在处理长数字时会导致精度丢失

### 最佳实践
1. **生成 Excel**：使用 openpyxl，手动设置单元格格式
2. **读取 Excel**：使用 dtype 参数，明确指定列类型
3. **文档说明**：在模板和文档中说明格式要求

### 适用场景
- 银行卡号（16-19位）
- 身份证号（18位）
- 订单号（长数字）
- 任何超过15位的数字
