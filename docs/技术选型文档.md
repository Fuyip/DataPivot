# 数枢 (DataPivot) 技术选型文档

## 1. 技术选型概述

### 1.1 选型原则
- **稳定性优先**: 选择成熟、稳定的技术栈，确保系统可靠运行
- **性能优先**: 处理大规模数据时保证高性能
- **开发效率**: 提高开发效率，降低维护成本
- **安全性**: 满足执法数据的安全要求
- **可扩展性**: 支持未来功能扩展

## 2. 前端技术选型

### 2.1 核心框架: Vue 3

**选择理由:**
- **渐进式框架**: 易于上手，学习曲线平缓
- **组合式API**: 更好的代码组织和逻辑复用
- **性能优异**: 虚拟DOM优化，响应式系统重构
- **生态完善**: 丰富的第三方库和工具链
- **TypeScript支持**: 可选的类型安全

**版本**: Vue 3.4+

### 2.2 UI组件库: Element Plus

**选择理由:**
- **组件丰富**: 提供60+高质量组件
- **企业级应用**: 适合后台管理系统
- **中文文档**: 完善的中文文档和社区支持
- **定制性强**: 支持主题定制
- **Vue 3原生支持**: 专为Vue 3设计

**备选方案**: Ant Design Vue (功能更丰富，但体积较大)

### 2.3 状态管理: Pinia

**选择理由:**
- **Vue 3官方推荐**: Vuex的继任者
- **TypeScript友好**: 完整的类型推断
- **轻量级**: 体积小，性能好
- **模块化**: 自动代码分割
- **DevTools支持**: 完善的调试工具

### 2.4 路由管理: Vue Router 4

**选择理由:**
- **Vue官方路由**: 与Vue深度集成
- **动态路由**: 支持权限控制
- **导航守卫**: 完善的路由拦截机制
- **懒加载**: 支持路由级代码分割

### 2.5 HTTP客户端: Axios

**选择理由:**
- **浏览器兼容性好**: 支持IE11+
- **拦截器机制**: 统一处理请求/响应
- **取消请求**: 支持请求取消
- **自动转换**: JSON数据自动转换
- **社区成熟**: 大量最佳实践

### 2.6 数据可视化

#### 2.6.1 ECharts 5
**用途**: 统计图表、关系图谱、地图可视化

**选择理由:**
- **功能强大**: 支持50+图表类型
- **性能优异**: Canvas渲染，支持大数据量
- **交互丰富**: 完善的交互功能
- **中文文档**: 百度开源，中文支持好

#### 2.6.2 D3.js (可选)
**用途**: 复杂的自定义可视化

**选择理由:**
- **灵活性高**: 完全自定义
- **社区丰富**: 大量示例和插件
- **数据驱动**: 强大的数据绑定能力

### 2.7 构建工具: Vite

**选择理由:**
- **极速启动**: 基于ESM的开发服务器
- **热更新快**: 毫秒级HMR
- **按需编译**: 只编译当前页面
- **生产优化**: 基于Rollup的生产构建
- **Vue官方推荐**: Vue团队核心成员开发

### 2.8 其他前端工具

| 工具 | 用途 | 选择理由 |

| **Day.js** | 日期处理 | 轻量级(2KB)，API与Moment.js兼容 |

## 3. 后端技术选型

### 3.1 核心框架: FastAPI

**选择理由:**
- **高性能**: 基于Starlette和Pydantic，性能接近Node.js和Go
- **异步支持**: 原生async/await支持
- **自动文档**: 自动生成OpenAPI文档
- **类型检查**: 基于Python类型提示
- **依赖注入**: 优雅的依赖注入系统
- **WebSocket支持**: 原生支持实时通信
- **现代化**: Python 3.6+特性

**版本**: FastAPI 0.110+

**性能对比**:
```
框架性能对比 (请求/秒)
FastAPI:    ~20,000
Flask:      ~3,000
Django:     ~2,000
```

### 3.2 ORM框架: SQLAlchemy

**选择理由:**
- **功能强大**: Python最成熟的ORM
- **灵活性高**: 支持原生SQL和ORM混用
- **异步支持**: SQLAlchemy 2.0支持异步
- **连接池**: 内置连接池管理
- **迁移工具**: 配合Alembic进行数据库迁移
- **多数据库支持**: 支持MySQL、PostgreSQL等

**版本**: SQLAlchemy 2.0+

### 3.3 数据处理: Pandas + NumPy

**选择理由:**
- **Pandas**:
  - 强大的数据清洗能力
  - 支持多种数据格式(CSV、Excel、SQL等)
  - 丰富的数据分析函数
  - 高效的数据转换

- **NumPy**:
  - 高性能数值计算
  - 多维数组支持
  - 线性代数运算
  - Pandas的底层依赖

### 3.4 异步任务: Celery + Redis

**选择理由:**
- **Celery**:
  - 成熟的分布式任务队列
  - 支持定时任务
  - 任务重试机制
  - 任务监控

- **Redis**:
  - 高性能消息队列
  - 支持持久化
  - 丰富的数据结构
  - 可作为缓存使用

**使用场景**:
- 大文件上传处理
- 批量数据导入
- 报告生成
- 定时数据同步

### 3.5 日志管理: Loguru

**选择理由:**
- **简单易用**: 零配置即可使用
- **功能丰富**: 日志轮转、压缩、邮件通知
- **性能好**: 异步日志写入
- **格式化**: 支持彩色输出和自定义格式
- **异常捕获**: 自动捕获异常堆栈

**配置示例**:
```python
from loguru import logger

logger.add(
    "logs/app_{time}.log",
    rotation="500 MB",
    retention="30 days",
    compression="zip",
    level="INFO"
)
```

### 3.6 数据验证: Pydantic

**选择理由:**
- **FastAPI内置**: 与FastAPI深度集成
- **类型安全**: 基于Python类型提示
- **数据验证**: 自动验证和转换
- **JSON Schema**: 自动生成JSON Schema
- **性能优异**: 使用Rust加速

## 4. 数据库技术选型

### 4.1 关系型数据库: MySQL 8.0

**选择理由:**
- **成熟稳定**: 使用最广泛的开源数据库
- **性能优异**: 8.0版本性能大幅提升
- **JSON支持**: 原生JSON数据类型
- **窗口函数**: 支持复杂分析查询
- **全文索引**: 支持中文全文搜索
- **事务支持**: ACID特性保证数据一致性
- **主从复制**: 支持高可用部署

**版本**: MySQL 8.0+

**配置建议**:
```ini
[mysqld]
# 字符集
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci

# InnoDB配置
innodb_buffer_pool_size=2G
innodb_log_file_size=512M
innodb_flush_log_at_trx_commit=2

# 连接数
max_connections=500

# 查询缓存
query_cache_type=0
```

### 4.2 备选方案: PostgreSQL

**优势**:
- JSON支持更强
- 复杂查询性能更好
- 支持更多数据类型
- 扩展性更强

**不选择原因**:
- 团队对MySQL更熟悉
- 运维成本较高
- 生态工具较少

### 4.3 缓存: Redis (可选)

**用途**:
- 会话存储
- 查询结果缓存
- 任务队列
- 实时数据

**版本**: Redis 6.0+

## 6. 部署技术选型

### 6.1 容器化: Docker

**选择理由:**
- 环境一致性
- 快速部署
- 资源隔离
- 易于扩展

**Docker Compose配置**:
```yaml
version: '3.8'
services:
  backend:
    build: ./backend
    ports:
      - "8000:8000"
    depends_on:
      - mysql
      - redis

  frontend:
    build: ./frontend
    ports:
      - "80:80"

  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_PASSWORD}
      MYSQL_DATABASE: datapivot

  redis:
    image: redis:6-alpine
```

### 6.2 Web服务器: Nginx

**用途**:
- 反向代理
- 静态文件服务
- 负载均衡
- SSL终止

### 6.3 ASGI服务器: Uvicorn + Gunicorn

**配置**:
```bash
gunicorn backend.main:app \
  --workers 4 \
  --worker-class uvicorn.workers.UvicornWorker \
  --bind 0.0.0.0:8000
```

### 6.4 进程管理: Supervisor

**用途**:
