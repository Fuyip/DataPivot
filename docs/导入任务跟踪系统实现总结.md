2026-03-09

## 功能概述
实现了完整的导入任务跟踪系统，允许用户管理每次导入操作，批量删除导入的数据，以及重新匹配未匹配的银行名称。

## 数据库变更

### 1. 新增表：import_task
位置：datapivot 系统数据库

```sql
CREATE TABLE `import_task` (
  `id` INT AUTO_INCREMENT PRIMARY KEY,
  `case_id` INT NOT NULL COMMENT '案件ID',
  `task_type` VARCHAR(50) NOT NULL COMMENT '任务类型：case_card/bank_statement',
  `file_name` VARCHAR(255) COMMENT '导入文件名',
  `total_count` INT DEFAULT 0 COMMENT '总记录数',
  `success_count` INT DEFAULT 0 COMMENT '成功数',
  `error_count` INT DEFAULT 0 COMMENT '失败数',
  `error_details` TEXT COMMENT '错误详情JSON',
  `created_by` INT NOT NULL COMMENT '创建人ID',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  INDEX idx_case_id (case_id),
  INDEX idx_task_type (task_type),
  INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 2. 修改表：case_card
为每个案件数据库的 case_card 表添加字段：

```sql
ALTER TABLE case_card
ADD COLUMN `import_task_id` INT DEFAULT NULL COMMENT '导入任务ID' AFTER `is_main`;

ALTER TABLE case_card
ADD INDEX idx_import_task_id (import_task_id);
```

## 核心功能说明

### 1. 导入任务记录
每次导入操作都会创建一个 import_task 记录，包含：
- 导入的文件名
- 总记录数、成功数、失败数
- 错误详情（JSON格式，最多保存50条）
- 创建人和创建时间

### 2. 银行名称匹配策略变更
**之前：** 如果卡号无法匹配银行名称，拒绝导入该记录
**现在：** 允许导入，bank_name 设为 NULL，用户可以后续使用"一键匹配银行"功能重新匹配

### 3. 批量删除导入数据
用户可以在导入任务列表中，选择某个任务，点击"删除数据"按钮，系统会删除该任务导入的所有银行卡记录。

### 4. 一键重新匹配银行
系统会查找所有 bank_name 为 NULL 的记录，尝试重新匹配银行名称，并更新成功匹配的记录。

## 权限控制

- **read** 权限：查看导入任务列表、查看错误详情
- **write** 权限：执行导入、一键重新匹配银行
- **admin** 权限：删除任务相关的所有银行卡

## 使用流程

### 导入流程
1. 用户点击"导入"按钮
2. 选择Excel文件上传
3. 系统创建导入任务记录
4. 逐行处理数据：
   - 检查卡号是否重复
   - 尝试匹配银行名称（失败不拒绝）
   - 验证卡类型
   - 插入数据并关联 import_task_id
5. 更新任务统计信息
6. 显示导入结果（成功数、失败数）
7. 如有错误，提供下载错误报告

### 任务管理流程
1. 用户点击"导入任务"按钮
2. 查看所有历史导入任务
3. 可以查看每个任务的错误详情
4. 可以删除某个任务导入的所有数据

### 重新匹配流程
1. 用户点击"一键匹配银行"按钮
2. 系统查找所有未匹配的银行卡
3. 逐个尝试重新匹配
4. 显示匹配结果统计

## 注意事项

1. **数据库迁移**：需要在所有案件数据库中执行 ALTER TABLE 语句添加 import_task_id 字段
2. **历史数据**：已存在的银行卡记录的 import_task_id 为 NULL
3. **错误详情限制**：每个任务最多保存50条错误记录
4. **删除操作**：删除任务相关的银行卡是不可逆操作，需要 admin 权限
