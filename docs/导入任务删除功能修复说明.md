# 导入任务删除功能修复说明

## 修复日期
2026-03-09

## 问题描述

用户反馈：**删除导入任务时返回 500 Internal Server Error**

后端错误：
```
pymysql.err.OperationalError: (1054, "Unknown column 'import_task_id' in 'where clause'")
[SQL: SELECT COUNT(*) FROM case_card WHERE import_task_id = %(task_id)s]
```

---

## 问题原因

### 根本原因：数据库表结构缺少字段

1. **字段缺失**：案件数据库的 `case_card` 表中没有 `import_task_id` 字段
2. **迁移未执行**：虽然存在迁移脚本 `migrations/add_import_task_support.sql`，但需要手动为每个案件数据库执行
3. **删除逻辑假设字段存在**：`delete_cards_by_task` 方法直接使用 `import_task_id` 字段，没有检查字段是否存在

### 为什么会缺少字段？

- `import_task_id` 字段是后来添加的功能
- 旧的案件数据库在创建时没有这个字段
- 需要通过数据库迁移为现有案件数据库添加此字段

---

## 解决方案

### 修复1：添加字段存在性检查

**文件**：`backend/services/import_task_service.py`

**修改位置**：第 110-145 行

**修改内容**：

```python
@staticmethod
def delete_cards_by_task(database_name: str, task_id: int) -> dict:
    """根据导入任务ID批量删除银行卡"""
    from backend.services.case_card_service import CaseCardService

    db = CaseCardService._get_case_db(database_name)

    try:
        # 检查表是否有 import_task_id 字段
        check_column_sql = """
            SELECT COUNT(*)
            FROM information_schema.COLUMNS
            WHERE TABLE_SCHEMA = DATABASE()
            AND TABLE_NAME = 'case_card'
            AND COLUMN_NAME = 'import_task_id'
        """
        has_import_task_id = db.execute(text(check_column_sql)).scalar() > 0

        if not has_import_task_id:
            # 如果表中没有 import_task_id 字段，无法按任务删除
            return {
                "deleted_count": 0,
                "expected_count": 0,
                "message": "该案件数据库表结构不支持按导入任务删除"
            }

        # 查询要删除的记录数
        count_sql = "SELECT COUNT(*) FROM case_card WHERE import_task_id = :task_id"
        count = db.execute(text(count_sql), {"task_id": task_id}).scalar()

        # 删除记录
        delete_sql = "DELETE FROM case_card WHERE import_task_id = :task_id"
        result = db.execute(text(delete_sql), {"task_id": task_id})
        db.commit()

        return {
            "deleted_count": result.rowcount,
            "expected_count": count
        }
    except Exception as e:
        db.rollback()
        raise ValueError(f"删除失败: {str(e)}")
    finally:
        db.close()
```

**说明**：
- 在执行删除前，先检查 `import_task_id` 字段是否存在
- 如果字段不存在，返回友好的提示信息，而不是抛出错误
- 如果字段存在，正常执行删除逻辑

## 其他案件数据库迁移

### 如何为其他案件数据库添加字段

如果其他案件也需要支持按导入任务删除功能，需要执行以下步骤：

#### 1. 查询案件数据库名称

```sql
SELECT id, case_code, database_name
FROM datapivot.cases
WHERE id = {case_id};
```

#### 2. 为该数据库添加字段

```sql
USE {database_name};

ALTER TABLE case_card
ADD COLUMN import_task_id INT DEFAULT NULL COMMENT '导入任务ID'
AFTER is_main;

ALTER TABLE case_card
ADD INDEX idx_import_task_id (import_task_id);
```

#### 3. 验证字段是否添加成功

```sql
SHOW COLUMNS FROM case_card LIKE 'import_task_id';
```

### 批量迁移脚本

如果需要为所有案件数据库添加字段，可以使用以下脚本：

```bash
#!/bin/bash
# 批量为所有案件数据库添加 import_task_id 字段

MYSQL_HOST="10.8.0.5"
MYSQL_USER="fuyip_net_gk"
MYSQL_PASS="Fuyip235813"

# 获取所有案件数据库名称
databases=$(mysql -h $MYSQL_HOST -u $MYSQL_USER -p$MYSQL_PASS -N -e \
  "SELECT database_name FROM datapivot.cases WHERE is_deleted = 0;")

# 为每个数据库添加字段
for db in $databases; do
  echo "处理数据库: $db"

  # 检查字段是否已存在
  exists=$(mysql -h $MYSQL_HOST -u $MYSQL_USER -p$MYSQL_PASS $db -N -e \
    "SHOW COLUMNS FROM case_card LIKE 'import_task_id';" | wc -l)

  if [ $exists -eq 0 ]; then
    echo "  添加 import_task_id 字段..."
    mysql -h $MYSQL_HOST -u $MYSQL_USER -p$MYSQL_PASS $db -e \
      "ALTER TABLE case_card ADD COLUMN import_task_id INT DEFAULT NULL COMMENT '导入任务ID' AFTER is_main;"

    echo "  添加索引..."
    mysql -h $MYSQL_HOST -u $MYSQL_USER -p$MYSQL_PASS $db -e \
      "ALTER TABLE case_card ADD INDEX idx_import_task_id (import_task_id);"

    echo "  完成"
  else
    echo "  字段已存在，跳过"
  fi
done

echo "所有数据库迁移完成"
```

### 为什么不在创建案件时自动添加字段？

这是一个好建议。应该在案件创建时的数据库初始化脚本中包含 `import_task_id` 字段。

**建议修改**：`sql/init/002_create_cases_tables.sql`

```sql
`import_task_id` INT DEFAULT NULL COMMENT '导入任务ID',

```

### 旧数据的 import_task_id 为什么是 NULL？

- 在添加字段之前导入的数据，`import_task_id` 为 NULL
- 这些数据无法通过导入任务删除功能删除
- 只能通过常规的单条删除或批量删除功能删除

### 如何处理字段不存在的情况？

代码已经处理了这种情况：
1. 检查字段是否存在
2. 如果不存在，返回友好提示而不是报错
3. 前端可以根据返回的 message 字段提示用户
