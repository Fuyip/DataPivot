### 问题 10：导入任务查询 500 错误（NaN 序列化）

**用户反馈**：
```
path.card_id: Input should be a valid integer, unable to parse string as an integer 获取导入任务失败。
```

**实际错误**：
```
500 Internal Server Error
ValueError: Out of range float values are not JSON compliant: nan
when serializing dict item 'card_no'
```

**根本原因**：
- 数据库中存储了包含 `nan` 值的历史导入任务错误详情
- `json.loads()` 将 `nan` 解析为 `float('nan')`
- FastAPI 返回响应时无法序列化 `float('nan')`

**解决方案**：
- 添加 `_clean_nan_values()` 方法递归清理 NaN 值
- 在查询导入任务时自动清理历史数据中的 NaN 值
- 将 NaN 值替换为字符串 "未知"

**修改文件**：
- `backend/services/import_task_service.py`（添加清理方法和调用）

**文档**：
- `docs/导入任务查询NaN序列化修复说明.md`

## 技术要点

### 1. NaN 值处理

**问题**：
- pandas 读取空单元格为 `nan`
- `json.loads()` 将 JSON 中的 `NaN` 解析为 `float('nan')`
- `json.dumps()` 无法序列化 `float('nan')`

**解决方案**：
```python
import math

def _clean_nan_values(obj):
    if isinstance(obj, dict):
        return {k: _clean_nan_values(v) for k, v in obj.items()}
    elif isinstance(obj, list):
        return [_clean_nan_values(item) for item in obj]
    elif isinstance(obj, float) and math.isnan(obj):
        return "未知"
    else:
        return obj
```

### 2. 动态字段检查

**问题**：
- 不同案件数据库的表结构可能不一致
- 直接使用字段会导致 SQL 错误

**解决方案**：
```python
check_column_sql = """
    SELECT COUNT(*)
    FROM information_schema.COLUMNS
    WHERE TABLE_SCHEMA = DATABASE()
    AND TABLE_NAME = 'case_card'
    AND COLUMN_NAME = 'import_task_id'
"""
has_import_task_id = db.execute(text(check_column_sql)).scalar() > 0

if not has_import_task_id:
    return {"message": "该案件数据库表结构不支持按导入任务删除"}
```

### 已实现的兼容性措施

1. **NaN 值清理**
   - 不修改数据库原始数据
   - 在查询时动态清理

2. **字段检查**

- 在字段添加之前导入的数据，`import_task_id` 为 NULL
- 这些数据无法通过导入任务删除功能删除
