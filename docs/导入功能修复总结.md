## 技术要点

### 1. pandas 类型推断问题

pandas 读取 Excel 时会自动推断类型，长数字会被转换为 float64，导致精度丢失。

**解决方法**：使用 `dtype` 参数明确指定列类型
```python
df = pd.read_excel(file, dtype={'卡号': str})
```

### 2. 宽容的导入策略

导入功能采用"宽容"策略，尽量不拒绝数据：
- 银行名称未匹配 → 设为 NULL，继续导入
- 卡类型无效 → 设为 NULL，继续导入
- 只有卡号缺失或重复时才拒绝

### 3. 向后兼容性设计

系统设计考虑了向后兼容：
- 动态检查数据库字段是否存在
- 自动适配不同版本的数据库
- 不强制要求立即迁移
