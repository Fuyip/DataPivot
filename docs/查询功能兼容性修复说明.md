# 查询功能兼容性修复说明

## 修复日期
2026-03-09

## 问题描述

导入功能测试时发现：
- 导入请求返回 200 OK（成功）
- 但查询列表时返回 500 Internal Server Error（失败）

**错误日志**：
```
INFO: 127.0.0.1:58391 - "GET /api/v1/cases/22/case-cards?page=1&page_size=20..." 500 Internal Server Error
```

## 问题原因

查询方法（`get_case_cards` 和 `get_case_card`）在 SELECT 语句中包含了 `import_task_id` 字段，但数据库可能没有这个字段，导致查询失败。

**问题代码**：
```python
query_sql = """
    SELECT id, case_no, source, user_id, card_no, bank_name, card_type,
           add_date, batch, is_in_bg, is_main, import_task_id
    FROM case_card
    WHERE ...
"""
```

如果数据库中 case_card 表没有 import_task_id 字段，这个查询会报错。

---

## 解决方案

### 修复1：列表查询兼容性

**文件**：`backend/services/case_card_service.py`

**方法**：`get_case_cards()`

**修改位置**：第55-86行

**修改内容**：
```python
# 检查表是否有 import_task_id 字段
check_column_sql = """
    SELECT COUNT(*)
    FROM information_schema.COLUMNS
    WHERE TABLE_SCHEMA = DATABASE()
    AND TABLE_NAME = 'case_card'
    AND COLUMN_NAME = 'import_task_id'
"""
has_import_task_id = db.execute(text(check_column_sql)).scalar() > 0

# 根据字段是否存在构建查询
if has_import_task_id:
    query_sql = f"""
        SELECT id, case_no, source, user_id, card_no, bank_name, card_type,
               add_date, batch, is_in_bg, is_main, import_task_id
        FROM case_card
        WHERE {where_sql}
        ORDER BY add_date DESC
        LIMIT :limit OFFSET :offset
    """
else:
    query_sql = f"""
        SELECT id, case_no, source, user_id, card_no, bank_name, card_type,
               add_date, batch, is_in_bg, is_main
        FROM case_card
        WHERE {where_sql}
        ORDER BY add_date DESC
        LIMIT :limit OFFSET :offset
    """
```

## 兼容性策略

### 写入操作（INSERT）
- 动态检查字段是否存在
- 存在则插入 import_task_id
- 不存在则跳过该字段

### 读取操作（SELECT）
- 动态检查字段是否存在
- 存在则查询 import_task_id
- 不存在则不查询该字段

### 更新操作（UPDATE）
- 不涉及 import_task_id 字段
- 无需特殊处理

### 删除操作（DELETE）
- 不涉及 import_task_id 字段
- 无需特殊处理

## 性能考虑

### 字段检查的性能影响

每次查询都检查字段是否存在会有性能开销，但：

1. **查询很快**：information_schema 查询通常在毫秒级
2. **可以缓存**：可以在应用启动时检查一次，缓存结果
3. **临时方案**：这是过渡期的兼容方案，最终所有数据库都会迁移
