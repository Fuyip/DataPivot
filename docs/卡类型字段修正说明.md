# 卡类型字段修正说明

## 修正日期
2026-03-09

## 问题描述

用户指出：卡类型字段应该存储 `d_label`（显示名称），而不是 `d_value`（代号）。

**错误示例**：
```
card_type = "c15q"  // 错误：存储的是 value
```

**正确示例**：
```
card_type = "疑似员工卡"  // 正确：存储的是 label
```

---

## 数据库设计

### sys_dict 表结构

| 字段 | 说明 | 示例 |
|------|------|------|
| d_type | 字典类型 | card_type |
| d_value | 代号（内部使用） | c15q |
| d_label | 显示名称（存储到 case_card） | 疑似员工卡 |

### case_card 表的 card_type 字段

**应该存储**：`d_label`（如"疑似员工卡"）
**不应该存储**：`d_value`（如"c15q"）

---

## 修正方案

### 修改1：导入逻辑

**文件**：`backend/services/case_card_service.py`

**修改前**：
```python
# 建立 label 到 value 的映射
card_type_map = {ct['label']: ct['value'] for ct in card_types}

# 将 label 转换为 value 存储
card_type_value = card_type_map.get(card_type_label)
card_data["card_type"] = card_type_value
```

**修改后**：
```python
# 建立 value 到 label 的映射
value_to_label_map = {ct['value']: ct['label'] for ct in card_types}
label_to_label_map = {ct['label']: ct['label'] for ct in card_types}

# 处理卡类型：将 value 转换为 label，或保持 label 不变
if card_type_input in label_to_label_map:
    card_type_label = card_type_input  # 已经是 label
elif card_type_input in value_to_label_map:
    card_type_label = value_to_label_map[card_type_input]  # 转换 value 为 label
else:
    card_type_label = None  # 无效

card_data["card_type"] = card_type_label
```

**改进点**：
1. 支持输入 label 格式（如"疑似员工卡"）→ 直接存储
2. 支持输入 value 格式（如"c15q"）→ 转换为 label 后存储
3. 最终存储到数据库的都是 label
