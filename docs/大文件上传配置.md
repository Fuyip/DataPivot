## 注意事项

### 文件大小限制
- **小于 20GB**：正常上传
- **等于或大于 20GB**：会显示错误信息

### 超时设置
- 前端超时：10 分钟
- 后端保持连接：5 分钟
- 如果上传时间超过 10 分钟，需要进一步增加超时时间

### 性能建议
- 对于超大文件（> 5GB），建议使用压缩格式
- 网络不稳定时，建议分批上传
- 上传过程中不要关闭浏览器标签页

## 故障排查

### 如果仍然上传失败

1. **检查后端日志**：
   - 查看后端终端的错误信息
   - 确认是否有内存或磁盘空间不足的错误

2. **检查浏览器控制台**：
   - 打开开发者工具（F12）
   - 查看 Console 和 Network 标签页的错误信息

3. **测试小文件**：
   - 先尝试上传一个小文件（< 100MB）
   - 确认基本功能正常

4. **检查磁盘空间**：
   ```bash
   df -h
   ```
   确保有足够的磁盘空间存储上传的文件

### 如果需要支持更大的文件

修改以下配置：

1. `backend/main.py` 中的 `app.state.max_body_size`
2. `frontend/src/api/bankStatement.js` 中的 `maxContentLength` 和 `maxBodyLength`
3. `frontend/src/api/bankStatement.js` 中的 `timeout`

## 系统资源要求

上传 20GB 文件需要：
- 至少 20GB 可用磁盘空间
- 稳定的网络连接
- 足够的内存（建议至少 4GB 可用内存）
