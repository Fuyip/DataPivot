# 测试数据科学计数法问题修复说明

## 问题描述

在使用 pandas 生成测试数据时，Excel 会将长数字（如银行卡号）自动转换为科学计数法格式，导致导入时卡号格式错误。

**错误示例**：
```
6217001234567890123 → 6.21700123456789e+18
```

## 问题原因

科学计数法问题有**两个来源**，需要分别修复：

### 问题1：Excel 文件生成时
- pandas 的 `to_excel()` 方法默认将长数字保存为数值类型
- Excel 对于超过15位的数字会自动使用科学计数法显示

### 问题2：Excel 文件读取时（更关键）
- pandas 的 `read_excel()` 方法默认将数字列识别为 float64 类型
- **即使 Excel 中是文本格式，pandas 也会尝试转换为数字**
- 导入时读取到的是科学计数法，无法匹配银行

## 解决方案

需要在**两个地方**修复：

### 方案1：生成 Excel 文件时（测试数据）

使用 `openpyxl` 库直接生成 Excel 文件，并将卡号列设置为文本格式。

#### 修复代码

```python
from openpyxl import Workbook

wb = Workbook()
ws = wb.active

# 添加表头
headers = ["卡号", "卡类型", "卡主姓名", "用户ID", "批次"]
ws.append(headers)

# 添加数据
test_data = [
    ["6217001234567890123", "c15q", "张三", "external_user_001", 1],
    # ... 更多数据
]

for row_data in test_data:
    ws.append(row_data)

# 关键：设置卡号列（A列）为文本格式
for row in range(2, ws.max_row + 1):
    cell = ws.cell(row=row, column=1)
    cell.number_format = '@'  # @ 表示文本格式

wb.save("测试数据.xlsx")
```

### 方案2：读取 Excel 文件时（导入服务）

在后端导入服务中，使用 `dtype` 参数指定卡号列为字符串类型。

#### 修复代码

文件：`backend/services/case_card_service.py`

```python
# 读取Excel文件，指定卡号列为字符串类型，防止科学计数法
df = pd.read_excel(BytesIO(file_content), dtype={'卡号': str})
```

**关键点**：
- `dtype={'卡号': str}` 强制将卡号列读取为字符串
- 即使 Excel 中是数字格式，也会保持原始值
- 避免 pandas 自动转换为 float64 导致精度丢失

#### 验证效果

```python
# 不指定 dtype（错误）
df1 = pd.read_excel(file)
# 结果: 6.21700123456789e+18 (float64)

# 指定 dtype={'卡号': str}（正确）
df2 = pd.read_excel(file, dtype={'卡号': str})
# 结果: 6217001234567890123 (str)
```

## 技术细节

### 双重修复的必要性

| 修复位置 | 作用 | 是否必需 |
|---------|------|---------|
| Excel 生成（openpyxl） | 确保文件中是文本格式 | 推荐但非必需 |
| Excel 读取（dtype参数） | 确保读取时不转换为数字 | **必需** ⭐ |

**结论**：即使 Excel 文件中卡号是文本格式，pandas 读取时仍会尝试转换为数字。因此 `dtype={'卡号': str}` 参数是**必需的**。

### 为什么使用 openpyxl 而不是 pandas（生成文件）

| 特性 | pandas | openpyxl |
|------|--------|----------|
| 数字格式控制 | 有限 | 完全控制 |
| 单元格格式 | 自动推断 | 手动设置 |
| 文本格式 | 困难 | 简单 |
| 适用场景 | 数据分析 | Excel 文件生成 |

### pandas dtype 参数说明（读取文件）

```python
# dtype 参数可以指定列的数据类型
df = pd.read_excel(file, dtype={
    '卡号': str,      # 字符串类型
    '批次': int,      # 整数类型
    '金额': float     # 浮点数类型
})
```

**优点**：
- 避免自动类型推断导致的精度丢失
- 保持原始数据格式
- 提高读取性能（跳过类型推断）

### 文本格式代码说明

```python
cell.number_format = '@'
```

- `@` 是 Excel 的文本格式代码
- 设置后，单元格内容会被视为文本
- 即使是数字，也不会转换为科学计数法

## 其他注意事项

### 1. 导入模板

如果用户使用下载的模板填写数据，需要确保模板中的卡号列也是文本格式。

**建议**：
- 在模板生成时设置卡号列为文本格式
- 在模板中添加说明，提示用户卡号列应为文本格式

### 2. 用户手动输入

如果用户手动在 Excel 中输入卡号：
- 在卡号前加单引号：`'6217001234567890123`
- 或先设置单元格格式为文本，再输入

### 3. CSV 格式

如果使用 CSV 格式：
- CSV 不会有科学计数法问题
- 但需要注意编码问题（建议使用 UTF-8）
