# 案件数据库创建指南

## 概述

DataPivot 系统采用多数据库架构，每个案件使用独立的数据库。所有案件数据库的表结构与 `fx_test` 数据库保持一致。

**✨ 重要更新**: 系统已实现自动化！创建案件时会自动创建数据库并初始化所有 76 个表结构，无需手动操作。

## 数据库结构说明

### 系统数据库
- **datapivot**: 系统核心数据库，存储用户认证、案件管理等系统配置

### 案件数据库
- **命名规则**: `{案件名称}_{案件编号}` （如 `新案件_CASE001`）
- **表结构**: 与 `fx_test` 数据库完全一致（76个表）
- **字符集**: utf8mb4
- **排序规则**: utf8mb4_unicode_ci
- **自动创建**: ✅ 通过 API 创建案件时自动完成

## 推荐方式：通过 API 创建案件（自动化）

**这是最简单的方式！** 系统会自动完成所有操作。

### 通过前端界面创建

1. 登录系统
2. 进入"案件管理"页面
3. 点击"创建案件"按钮
4. 填写案件信息
5. 点击"确定"

**系统自动完成**:
- ✅ 创建案件记录
- ✅ 创建案件数据库
- ✅ 初始化 76 个表结构
- ✅ 分配创建者管理权限

### 通过 API 创建

```bash
curl -X POST http://localhost:8000/api/v1/cases \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "case_name": "新案件",
    "case_code": "CASE001",
    "description": "案件描述"
  }'
```

**返回示例**:
```json
{
  "code": 200,
  "message": "案件创建成功",
  "data": {
    "id": 1,
    "case_name": "新案件",
    "case_code": "CASE001",
    "database_name": "新案件_CASE001",
    "description": "案件描述",
    "status": "active"
  }
}
```

**数据库 `新案件_CASE001` 已自动创建，包含 76 个表！**

## 手动创建方式（不推荐）

如果需要手动创建（例如批量导入、脚本操作等），可以使用以下方法：

### 方法一：使用模板文件

已为您准备好案件数据库模板文件：`sql/schema/case_template.sql`

```bash
# 1. 创建新的案件数据库（数据库名格式：{案件名称}_{案件编号}）
mysql -uroot -p -e "CREATE DATABASE 新案件_CASE001 CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 2. 导入表结构
mysql -uroot -p 新案件_CASE001 < sql/schema/case_template.sql

# 3. 在系统中注册案件（见下文）
```

### 方法二：使用 Python 脚本

如果需要重新从 `fx_test` 导出最新结构：

```bash
# 激活虚拟环境
source venv/bin/activate

# 运行导出脚本
python export_fx_test_schema.py

# 使用生成的模板创建新数据库
mysql -uroot -p case_xxx < sql/schema/case_template.sql
```

## 在系统中注册案件

创建案件数据库后，需要在 `datapivot` 系统数据库中注册：

```sql
USE datapivot;

INSERT INTO cases (
    case_name,
    case_code,
    database_name,
    description,
    status,
    created_by
) VALUES (
    '新案件',
    'CASE001',
    '新案件_CASE001',  -- 数据库名格式：{案件名称}_{案件编号}
    '案件描述',
    'active',
    1  -- 创建人用户ID
);
```

或通过系统 API 创建（推荐）：

```bash
curl -X POST http://localhost:8000/api/v1/cases \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "case_name": "新案件",
    "case_code": "CASE001",
    "description": "案件描述"
  }'
```

**注意**: API 会自动生成数据库名称为 `新案件_CASE001`

## 注意事项

1. **数据库命名**: 系统自动使用 `{案件名称}_{案件编号}` 格式命名数据库
2. **字符集**: 必须使用 utf8mb4，支持完整的 Unicode 字符
3. **权限配置**: 确保应用用户有访问新数据库的权限
4. **备份**: 定期备份案件数据库
5. **结构同步**: 如果 fx_test 结构有更新，需要重新导出模板

## 批量创建多个案件数据库

如果需要批量创建，可以使用脚本：

```bash
#!/bin/bash
# 批量创建案件数据库

CASES=("case_001" "case_002" "case_003")

for case_name in "${CASES[@]}"
do
    echo "创建数据库: $case_name"
    mysql -uroot -p -e "CREATE DATABASE $case_name CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
    mysql -uroot -p $case_name < sql/schema/case_template.sql
    echo "✓ $case_name 创建完成"
done
```

## 删除案件数据库

**警告**: 删除操作不可恢复，请谨慎操作！

```bash
# 1. 先备份（重要！）
mysqldump -uroot -p 新案件_CASE001 > backup_新案件_CASE001.sql

# 2. 从系统中删除案件记录
mysql -uroot -p datapivot -e "DELETE FROM cases WHERE database_name='新案件_CASE001';"

# 3. 删除数据库
mysql -uroot -p -e "DROP DATABASE 新案件_CASE001;"
```

## 故障排查

### 问题1: 导入模板时报错

```
ERROR 1064: You have an error in your SQL syntax
```

**解决方案**: 检查 MySQL 版本是否为 8.0+，确保字符集设置正确

### 问题2: 连接数据库失败

```
ERROR 2003: Can't connect to MySQL server
```

**解决方案**:
- 检查 MySQL 服务是否启动
- 检查 .env 配置文件中的数据库连接信息
- 确认用户权限

### 问题3: 表已存在

```
ERROR 1050: Table 'xxx' already exists
```

**解决方案**: 模板文件中已包含 `DROP TABLE IF EXISTS`，如果仍报错，手动删除数据库后重新创建
