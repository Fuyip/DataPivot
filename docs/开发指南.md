# 数枢 (DataPivot) 开发指南

## 1. 开发环境搭建

### 1.1 系统要求
- **操作系统**: macOS / Linux / Windows 10+
- **Python**: 3.9+
- **Node.js**: 16+
- **MySQL**: 8.0+
- **Git**: 2.0+

### 1.2 开发工具推荐
- **IDE**: VS Code / PyCharm Professional
- **数据库工具**: Navicat / DBeaver
- **API测试**: Postman / Insomnia
- **终端**: iTerm2 (macOS) / Windows Terminal

### 1.3 环境配置步骤

#### 1.3.1 克隆项目
```bash
git clone <repository_url>
cd DataPivot
```

#### 1.3.2 配置Python环境
```bash
# 创建虚拟环境
python3 -m venv venv

# 激活虚拟环境
# macOS/Linux:
source venv/bin/activate
# Windows:
venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt
```

#### 1.3.3 配置数据库
```bash
# 登录MySQL
mysql -u root -p

# 创建系统核心数据库
CREATE DATABASE datapivot CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 创建案件专用数据库（示例）
CREATE DATABASE case_20240301 CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 创建用户（可选）
CREATE USER 'datapivot'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON datapivot.* TO 'datapivot'@'localhost';
GRANT ALL PRIVILEGES ON `case_%`.* TO 'datapivot'@'localhost';
FLUSH PRIVILEGES;
```

**数据库架构说明**：
- `datapivot` 数据库：存储系统核心配置（用户认证、案件管理、系统配置）
- 案件专用数据库：每个案件独立数据库，数据库名称为案件代码，表结构参考 `fx_test` 数据库

#### 1.3.4 配置环境变量
```bash
# 复制环境变量模板
cp .env.example .env

# 编辑.env文件
vim .env
```

`.env` 文件内容：
```env
DEBUG_MODE=true

# 系统核心数据库配置
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=your_password
MYSQL_DB=datapivot

HAIJING_API_KEY=your_api_key
HAIJING_BASE_URL=https://api.example.com

SECRET_KEY=your-secret-key-change-this-in-production
```

**注意**：案件专用数据库连接需要在代码中动态创建，参考 `docs/DATABASE_DEVELOPMENT.md` 文档。

#### 1.3.5 初始化数据库表
```bash

```

## 4. 开发流程

### 4.1 功能开发流程

1. **创建功能分支**
```bash
git checkout -b feature/fund-trace-analysis
```

2. **编写代码**
   - 后端：编写Service → Model → API
   - 前端：编写API → Store → Component → View

3. **编写测试**
```python
# tests/test_bank.py
def test_fund_trace():
    result = bank_service.fund_trace("6222021234567890", depth=3)
    assert len(result['nodes']) > 0
```

4. **本地测试**
```bash
# 后端测试
pytest tests/

# 前端测试
npm run test
```

5. **提交代码**
```bash
git add .
git commit -m "feat(bank): 添加资金穿透分析功能"
git push origin feature/fund-trace-analysis
```

6. **创建Pull Request**

7. **代码审查**

8. **合并到develop分支**

### 4.2 Bug修复流程

1. **创建修复分支**
```bash
git checkout -b fix/person-query-error
```

2. **定位问题**
   - 查看日志
   - 复现bug
   - 调试代码

3. **修复并测试**

4. **提交代码**
```bash
git commit -m "fix(person): 修复人员查询空指针异常"
```

5. **合并到develop分支**

## 8. 常见问题

### 8.1 数据库连接失败
```bash

mysql.server status

# 检查端口是否被占用
lsof -i :3306

# 检查.env配置是否正确
```

```bash

pip cache purge

pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# npm使用淘宝镜像
```
