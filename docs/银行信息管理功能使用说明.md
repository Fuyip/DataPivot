# 银行信息管理功能使用说明

## 功能概述

银行信息管理功能用于维护系统中的银行卡 BIN 码库和银行名称映射表，支持管理员提交变更申请，超级管理员审批后执行变更。

## 访问路径

系统菜单 → 银行信息管理

## 权限说明

- **普通用户**: 无权访问此功能
- **管理员 (admin)**: 可以查看数据、提交变更申请
- **超级管理员 (super_admin)**: 拥有管理员所有权限 + 审批变更申请 + 直接修改数据

## 功能模块

### 1. BIN 码管理

#### 1.1 查看 BIN 码列表

- 支持按 BIN 码、银行名称、BIN 长度、卡长度搜索
- 分页展示所有 BIN 码记录

#### 1.2 新增 BIN 码

**管理员操作流程:**
1. 点击"新增"按钮
2. 填写表单:
   - BIN 码: 输入数字，系统自动计算 BIN 长度
   - 卡长度: 选择 16/17/18/19 位
   - 银行名称: 输入银行名称
   - 变更原因: 说明新增原因
3. 点击"提交申请"
4. 等待超级管理员审批

**超级管理员操作流程:**
1. 点击"新增"按钮
2. 填写表单（同上）
3. 可选择:
   - 勾选"直接执行": 立即添加到数据库
   - 不勾选: 提交申请等待审批（自己审批）

#### 1.3 修改 BIN 码

**操作流程:**
1. 点击记录后的"编辑"按钮
2. 修改需要变更的字段
3. 填写变更原因
4. 管理员: 提交申请等待审批
5. 超级管理员: 可选择直接执行或提交申请

#### 1.4 删除 BIN 码

**管理员操作流程:**
1. 点击记录后的"删除"按钮
2. 确认删除
3. 输入删除原因
4. 点击"提交申请"
5. 等待超级管理员审批

**超级管理员操作流程:**
1. 点击记录后的"删除"按钮
2. 确认删除
3. 输入删除原因（可选）
4. 选择:
   - "直接删除": ��即从数据库删除
   - "提交申请": 提交删除申请

#### 1.5 导出数据

- 点击"导出"按钮
- 导出当前筛选条件下的所有 BIN 码数据为 Excel 文件

#### 1.6 下载模板

- 点击"下载模板"按钮
- 下载 BIN 码导入模板（用于批量导入）

### 2. 银行名称映射管理

#### 2.1 查看映射列表

- 支持按原始银行名称、标准银行名称、系统标识搜索
- 分页展示所有映射记录

#### 2.2 新增映射

**操作流程:**
1. 点击"新增"按钮
2. 填写表单:
   - 原始银行名称: 输入原始名称
   - 标准银行名称: 输入标准名称
   - 系统标识: 默认为 'jz'
   - 变更原因: 说明新增原因
3. 管理员: 提交申请
4. 超级管理员: 可选��直接执行或提交申请

#### 2.3 修改映射

操作流程同 BIN 码修改

#### 2.4 删除映射

操作流程同 BIN 码删除

#### 2.5 导出和模板

功能同 BIN 码管理

### 3. 变更管理

#### 3.1 待审批列表（仅超级管理员可见）

显示所有待审批的变更申请，包括:
- 申请 ID
- 表类型（BIN 码库/银行映射）
- 变更类型（新增/修改/删除）
- 申请人
- 申请时间
- 变更原因

**审批操作:**

1. **批准申请**
   - 点击"批准"按钮
   - 输入审批意见（可选）
   - 点击"确定"
   - 系统自动执行变更并更新状态

2. **拒绝申请**
   - 点击"拒绝"按钮
   - 输入审批意见（建议说明拒绝原因）
   - 点击"确定"
   - 申请状态更新为"已拒绝"

3. **查看详情**
   - 点击"详情"按钮
   - 查看变更前后的数据对比
   - 查看变更原因

#### 3.2 变更历史

显示所有变更记录，包括:
- 待审批
- 已批准
- 已拒绝
- 已执行

**筛选功能:**
- 按状态筛选
- 按表类型筛选
- 按变更类型筛选

**操作:**
- 查看详情: 查看变更的详细信息和数据对比
- 撤销申请: 管理员可以撤销自己提交的待审批申请

## 使用场景示例

### 场景 1: 管理员新增 BIN 码

1. 管理员登录系统
2. 进入"银行信息管理" → "BIN 码管理"
3. 点击"新增"
4. 填写:
   - BIN 码: 622202
   - BIN 长度: 6（自动计算）
   - 卡长度: 16
   - 银行名称: 中国工商银行
   - 变更原因: 新增工商银行 BIN 码
5. 点击"提交申请"
6. 系统提示"申请已提交，等待审批"
7. 超级管理员收到待审批通知
8. 超级管理员进入"变更管理" → "待审批列表"
9. 查看申请详情，确认无误后点击"批准"
10. 系统执行变更，BIN 码添加到数据库

### 场景 2: 超级管理员直接修改

1. 超级管理员登录系统
2. 进入"银行信息管理" → "银行名称映射"
3. 找到需要修改的记录，点击"编辑"
4. 修改标准银行名称
5. 勾选"直接执行（跳过审批）"
6. 点击"确定"
7. 系统立即执行变更，无需审批

### 场景 3: 管理员撤销申请

1. 管理员提交了一个变更申请
2. 发现填写错误，需要撤销
3. 进入"变更管理" → "变更历史"
4. 找到自己提交的待审批申请
5. 点击"撤销"
6. 确认撤销
7. 申请被删除

## 注意事项

1. **权限限制**
   - 管理员只能提交申请，不能直接修改数据
   - 管理员只能撤销自己提交的待审批申请
   - 超级管理员可以选择直接执行或提交申请

2. **数据验证**
   - BIN 码必须是纯数字
   - BIN 长度会根据输入自动计算
   - 卡长度必须在 16-19 位之间
   - 所有必填字段不能为空

3. **审批流程**
   - 变更申请一旦被批准，会立即执行
   - 批准操作是事务性的，如果执行失败会自动回滚
   - 已批准或已拒绝的申请不能再次审批

4. **数据安全**
   - 所有变更都有完整的审计日志
   - 可以查看变更前后的数据对比
   - 记录申请人、审批人、时间等信息

5. **最佳实践**
   - 提交申请时请详细填写变更原因
   - 审批时建议填写审批意见
   - 定期查看变更历史，了解数据变更情况
   - 重要变更建议先提交申请，经过审批后执行

## 常见问题

**Q: 为什么我看不到"银行信息管理"菜单？**
A: 此功能仅对管理员和超级管理员开放，普通用户无权访问。

**Q: 管理员可以直接修改数据吗？**
A: 不可以。管理员只能提交变更申请，需要超级管理员审批后才能执行。

**Q: 超级管理员必��审批自己的申请吗？**
A: 不需要。超级管理员可以选择"直接执行"，跳过审批流程。

**Q: 如何查看我提交的申请状态？**
A: 进入"变更管理" → "变更历史"，可以查看所有申请的状态。

**Q: 申请被拒绝后可以重新提交吗？**
A: 可以。被拒绝的申请不会影响重新提交新的申请。

**Q: 批准申请后可以撤销吗？**
A: 不可以。一旦批准并执行，变更就已经生效，无法撤销。如需恢复，需要提交新的变更申请。
