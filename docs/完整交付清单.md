## 一、数据库变更

### 1.1 新增表

**表名**: `datapivot.import_task`

**用途**: 记录每次导入操作的详细信息

**字段**:
- `id` - 任务ID（主键）
- `case_id` - 案件ID
- `task_type` - 任务类型（case_card/bank_statement）
- `file_name` - 导入文件名
- `total_count` - 总记录数
- `success_count` - 成功数
- `error_count` - 失败数
- `error_details` - 错误详情（JSON）
- `created_by` - 创建人ID
- `created_at` - 创建时间

**索引**:
- PRIMARY KEY (`id`)
- INDEX `idx_case_id` (`case_id`)
- INDEX `idx_task_type` (`task_type`)
- INDEX `idx_created_at` (`created_at`)

### 1.2 修改表

**表名**: `{case_database}.case_card`

**新增字段**:
- `import_task_id` INT DEFAULT NULL COMMENT '导入任务ID'

**新增索引**:
- INDEX `idx_import_task_id` (`import_task_id`)

## 二、后端实现

### 2.1 新增文件

#### backend/models/import_task.py
- ImportTask 模型类
- 用于 ORM 操作 import_task 表

#### backend/services/import_task_service.py
- `get_import_tasks()` - 获取导入任务列表（分页）
- `get_import_task()` - 获取单个导入任务详情
- `delete_cards_by_task()` - 根据任务ID删除所有相关银行卡

### 2.2 修改文件

#### backend/services/case_card_service.py

**修改的方法**:
- `import_from_template()` - 重构导入逻辑
  - 添加 case_id 和 user_id 参数
  - 创建 ImportTask 记录
  - 银行名称匹配失败不再拒绝导入，设为 NULL
  - 保存 import_task_id 到每条记录
  - 更新任务统计信息

- `get_case_cards()` - 查询时包含 import_task_id 字段
- `get_case_card()` - 查询时包含 import_task_id 字段

**新增的方法**:
- `rematch_unmatched_banks()` - 重新匹配所有 bank_name 为 NULL 的记录
  - 查询未匹配的卡
  - 逐个尝试匹配
  - 更新成功匹配的记录
  - 返回匹配统计

#### backend/api/v1/case_cards.py

**修改的端点**:
- `POST /import` - 传递 case_id 和 user_id 给服务层

**新增的端点**:
- `GET /{case_id}/case-cards/import-tasks` - 获取导入任务列表
- `DELETE /{case_id}/case-cards/import-tasks/{task_id}` - 删除任务相关的所有银行卡
- `POST /{case_id}/case-cards/rematch-banks` - 一键重新匹配银行

## 六、核心功能实现

### 6.1 导入任务跟踪

**功能**:
- 每次导入创建任务记录
- 记录导入统计信息
- 保存错误详情（最多50条）

**实现位置**:
- 后端: `backend/services/case_card_service.py` - `import_from_template()`
- 前端: `frontend/src/components/CaseCard/CaseCardManager.vue` - 导入对话框

### 6.2 银行名称智能匹配

**功能**:
- 根据卡号 BIN 码自动匹配银行
- 匹配失败不拒绝导入（设为 NULL）
- 支持银行名称映射

**实现位置**:
- 后端: `backend/services/case_card_service.py` - `match_bank_name()`
- 数据库: `datapivot.bank_bin` + `datapivot.sy_bank`

### 6.3 一键重新匹配

**功能**:
- 查找所有未匹配的银行卡
- 批量重新匹配
- 返回匹配统计

**实现位置**:
- 后端: `backend/services/case_card_service.py` - `rematch_unmatched_banks()`
- 前端: `frontend/src/components/CaseCard/CaseCardManager.vue` - "一键匹配银行"按钮

### 6.4 批量删除任务数据

**功能**:
- 根据任务ID删除所有相关银行卡
- 需要 admin 权限
- 返回删除统计

**实现位置**:
- 后端: `backend/services/import_task_service.py` - `delete_cards_by_task()`
- 前端: `frontend/src/components/CaseCard/CaseCardManager.vue` - "删除数据"按钮
