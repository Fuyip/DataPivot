# 卡类型必填验证修复说明

## 修复日期
2026-03-09

## 问题描述

用户反馈：**银行卡类型不能为空**

之前的实现中，卡类型字段是可选的（Optional），导入时如果卡类型为空或无效，会设为 NULL 并继续导入。这不符合业务需求。

---

## 业务需求

1. 卡类型字段必须填写
2. 导入时如果卡类型为空，拒绝导入该条记录
3. 导入时如果卡类型无效（既不是 label 也不是 value），拒绝导入该条记录

---

## 解决方案

### 修改1：后端 Schema 验证

**文件**：`backend/schemas/case_card.py`

**修改位置**：第10行

**修改内容**：
```python
# 修改前
card_type: Optional[str] = Field(None, description="卡类型")

# 修改后
card_type: str = Field(..., description="卡类型")
```

**说明**：将 `card_type` 从 Optional 改为必填字段，API 创建和更新时会自动验证。

---

### 修改2：导入时卡类型验证

**文件**：`backend/services/case_card_service.py`

**修改位置**：第341-368行

**修改内容**：
```python
# 处理卡类型：将value转换为label，或保持label不变
card_type_input = str(row.get('卡类型', '')).strip() if pd.notna(row.get('卡类型')) else None
card_type_label = None
if card_type_input:
    # 先检查是否是label（如"入款卡"）
    if card_type_input in label_to_label_map:
        card_type_label = card_type_input
    # 再检查是否是value（如"c15q"），需要转换为label
    elif card_type_input in value_to_label_map:
        card_type_label = value_to_label_map[card_type_input]
    else:
        # 既不是label也不是value，拒绝导入
        error_count += 1
        errors.append({
            "row": index + 2,
            "card_no": card_no,
            "error": f"无效的卡类型: {card_type_input}"
        })
        continue
else:
    # 卡类型为空，拒绝导入
    error_count += 1
    errors.append({
        "row": index + 2,
        "card_no": card_no,
        "error": "卡类型不能为空"
    })
    continue
```

**说明**：
1. 如果卡类型为空，记录错误并跳过该条记录
2. 如果卡类型无效，记录错误并跳过该条记录
3. 只有有效的卡类型才会继续导入

---

## 验证逻辑

### 卡类型验证规则

| 输入值 | 是否有效 | 处理方式 |
|--------|---------|---------|
| 空值（NULL/空字符串） | ❌ 无效 | 拒绝导入，错误："卡类型不能为空" |
| "疑似员工卡"（label） | ✅ 有效 | 直接使用 |
| "c15q"（value） | ✅ 有效 | 转换为 "疑似员工卡" |
| "invalid_type"（无效） | ❌ 无效 | 拒绝导入，错误："无效的卡类型: invalid_type" |
