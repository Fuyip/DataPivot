## 常见问题

### Q1: 导入时所有记录都失败

**可能原因**:
- 数据库连接问题
- case_card 表缺少 import_task_id 字段
- 卡类型字典数据缺失

**解决方法**:
```bash
# 1. 检查数据库连接
python verify_database.py

# 2. 检查表结构
mysql -u fuyip_net_gk -p'Fuyip235813' -h 10.8.0.5 -e "DESC test01_GXMLM.case_card"

# 3. 检查卡类型字典
mysql -u fuyip_net_gk -p'Fuyip235813' -h 10.8.0.5 -e "SELECT * FROM datapivot.sys_dict WHERE d_type='card_type'"
```

### Q2: 所有卡号都无法匹配银行

**可能原因**:
- bank_bin 表数据不完整
- sy_bank 表缺少映射关系

**解决方法**:
```bash
# 检查 bank_bin 表
mysql -u fuyip_net_gk -p'Fuyip235813' -h 10.8.0.5 -e "SELECT COUNT(*) FROM datapivot.bank_bin"

# 检查 sy_bank 表
mysql -u fuyip_net_gk -p'Fuyip235813' -h 10.8.0.5 -e "SELECT COUNT(*) FROM datapivot.sy_bank WHERE sys='jz'"
```

### Q3: 导入任务列表为空

**可能原因**:
- import_task 表不存在
- 导入时未创建任务记录

**解决方法**:
```bash
# 检查 import_task 表
mysql -u fuyip_net_gk -p'Fuyip235813' -h 10.8.0.5 -e "SELECT * FROM datapivot.import_task ORDER BY created_at DESC LIMIT 5"
```

### Q4: 前端按钮不显示

**可能原因**:
- 前端代码未更新
- 浏览器缓存

**解决方法**:
```bash
# 1. 重新构建前端
cd frontend
npm run build

# 2. 清除浏览器缓存
# 按 Ctrl+Shift+R (Windows) 或 Cmd+Shift+R (Mac) 强制刷新
```

### Q5: API 返回 404

**可能原因**:
- 后端路由未注册
- 后端服务未重启

**解决方法**:
```bash
# 重启后端服务
cd backend
uvicorn main:app --reload --port 8000
```
