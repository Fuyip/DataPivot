# 数据库连接问题说明

## 问题描述

运行 `verify_database.py` 时出现以下错误：

```
Access denied for user 'fuyip_net_gk'@'10.8.0.49' (using password: YES)
```

## 原因分析

这是一个**数据库权限配置问题**，不是代码问题。

- **你的 IP**: 10.8.0.49
- **数据库服务器**: 10.8.0.5
- **问题**: 数据库服务器只允许特定 IP 连接，你的 IP 不在允许列表中

## 解决方案

### 方案一：添加数据库访问权限（推荐）

联系数据库管理员，执行以下 SQL 命令：

```sql
-- 为你的 IP 添加访问权限
GRANT ALL PRIVILEGES ON *.* TO 'fuyip_net_gk'@'10.8.0.49' IDENTIFIED BY 'Fuyip235813';
FLUSH PRIVILEGES;

-- 或者允许所有 IP（不推荐，安全性较低）
GRANT ALL PRIVILEGES ON *.* TO 'fuyip_net_gk'@'%' IDENTIFIED BY 'Fuyip235813';
FLUSH PRIVILEGES;
```

### 方案二：通过后端服务验证（临时方案）

如果后端服务可以正常连接数据库，说明数据库本身是正常的，只是你的本地机器无法直接连接。

**验证步骤**：

1. **启动后端服务**
   ```bash
   cd backend
   uvicorn main:app --reload --port 8000
   ```

2. **检查后端是否能连接数据库**
   ```bash
   curl http://localhost:8000/health
   ```

   如果返回成功，说明后端可以连接数据库。

3. **使用 API 测试脚本验证功能**
   ```bash
   # 编辑脚本设置 TOKEN
   vim test_complex_import.py

   # 运行测试
   python test_complex_import.py
   ```

### 方案三：在服务器上运行验证脚本

如果你有服务器的 SSH 访问权限，可以在服务器上运行验证脚本：

```bash
# SSH 到服务器
ssh user@10.8.0.5

# 上传验证脚本
scp verify_database.py user@10.8.0.5:/tmp/

# 在服务器上运行
python /tmp/verify_database.py
```

## 常见问题

### Q1: 为什么后端可以连接，但我的机器不能？

**A**: 数据库服务器通常配置了 IP 白名单，只允许特定的 IP 连接。后端服务器的 IP 在白名单中，但你的本地机器 IP 不在。

### Q2: 这会影响系统功能吗？

**A**: 不会。这只影响你本地运行验证脚本，不影响系统的正常功能。后端服务可以正常连接数据库。

### Q3: 我应该怎么做？

**A**:
1. 如果你需要经常直接访问数据库，联系管理员添加你的 IP 权限
2. 如果只是偶尔需要，使用方案二通过后端 API 验证
3. 如果只是测试功能，直接使用前端界面测试即可
