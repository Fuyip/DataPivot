## 问题描述

用户报告：导入测试文件时，卡号依然显示为科学计数法，导致无法正确匹配银行名称。

```
第2行 - 卡号: 6.21700123456789e+18 - 无效的卡类型: c15q
第3行 - 卡号: 6.22848123456789e+18 - 无效的卡类型: c30
...
```

## 问题分析

虽然之前已经使用 openpyxl 生成了文本格式的 Excel 文件，但问题依然存在。经过分析发现：

**根本原因**：pandas 在读取 Excel 文件时，会自动进行类型推断，将长数字列识别为 float64 类型，导致精度丢失和科学计数法显示。

**关键发现**：即使 Excel 文件中卡号列是文本格式，pandas 的 `read_excel()` 方法仍会尝试将其转换为数字类型。

## 解决方案

### 修改文件：`backend/services/case_card_service.py`

**位置**：`import_from_template()` 方法第246行

**修改前**：
```python
df = pd.read_excel(BytesIO(file_content))
```

**修改后**：
```python
df = pd.read_excel(BytesIO(file_content), dtype={'卡号': str})
```

**关键参数**：`dtype={'卡号': str}` 强制将卡号列读取为字符串类型，避免自动类型推断。

## 技术要点

### 为什么需要双重修复

| 修复位置 | 作用 | 是否必需 |
|---------|------|---------|
| Excel 生成（openpyxl） | 确保文件中是文本格式 | 推荐但非必需 |
| Excel 读取（dtype参数） | 确保读取时不转换为数字 | **必需** ⭐ |

### pandas 类型推断问题

pandas 的 `read_excel()` 方法会：
1. 自动推断每列的数据类型
2. 对于数字列，尝试转换为 int64 或 float64
3. 超过15位的数字使用 float64 存储
4. float64 精度有限，导致精度丢失
5. 打印时显示为科学计数法

### dtype 参数的优势

使用 `dtype` 参数可以：
1. 跳过自动类型推断
2. 直接将单元格内容读取为指定类型
3. 保持原始值不变
4. 避免精度丢失
5. 提高读取性能

## 适用场景

这个修复方案适用于所有需要读取长数字的场景：
- 银行卡号（16-19位）
- 身份证号（18位）
- 订单号（长数字）
- 手机号（11位）
- 任何超过15位的数字

```python
wb = Workbook()

    ws.append(row_data)

    cell = ws.cell(row=row, column=1)
    cell.number_format = '@'

wb.save(output_path)
```

### 读取 Excel 文件
```python
import pandas as pd

# 指定长数字列为字符串类型

    '身份证号': str,
    '手机号': str
})
```
