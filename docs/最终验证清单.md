## 技术要点总结

### 1. pandas 类型推断
- pandas 会自动将长数字转换为 float64
- 必须使用 `dtype` 参数明确指定类型
- 即使 Excel 文件格式正确，读取时仍需指定

### 2. 数据库兼容性
- 使用 information_schema 动态检查字段
- 根据检查结果生成不同的 SQL
- 支持已迁移和未迁移的数据库

### 3. 数据验证策略
- 采用"宽容"策略，尽量不拒绝数据
- 银行名称未匹配 → NULL
- 卡类型无效 → NULL
- 只有卡号缺失或重复时才拒绝

### 4. 字段设计规范
- card_type 存储 d_label（显示名称）
- 不存储 d_value（内部代号）
- 便于查询和显示
