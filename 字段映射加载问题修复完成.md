# 字段映射加载问题修复完成

## 问题描述

前端页面显示"加载字段映射失败"，后端返回500错误。

## 根本原因

数据库中有4条 `import_field_mappings` 记录的 `field_type` 字段值为 `'n'`，但Pydantic Schema验证要求该字段必须是以下值之一：
- `str` - 字符串
- `float` - 浮点数
- `datetime` - 日期时间
- `card_no` - 卡号
- `tag` - 收付标志
- `none` - 不处理
- `source` - 来源标记

## 错误详情

```
pydantic_core._pydantic_core.ValidationError: 1 validation error for FieldMapping
field_type
  Value error, 字段类型必须是以下之一: str, float, datetime, card_no, tag, none, source
  [type=value_error, input_value='n', input_type=str]
```

## 修复方案

将数据库中所有无效的 `field_type` 值修改为 `'none'`。

### 受影响的记录

```
ID=1,  template_id=1, data_type=人员信息,     db_field_name=source, field_type='n' -> 'none'
ID=24, template_id=1, data_type=账户信息,     db_field_name=source, field_type='n' -> 'none'
ID=68, template_id=1, data_type=子账户信息,   db_field_name=source, field_type='n' -> 'none'
ID=80, template_id=1, data_type=强制措施信息, db_field_name=source, field_type='n' -> 'none'
```

### 执行的SQL

```sql
UPDATE import_field_mappings
SET field_type = 'none'
WHERE field_type NOT IN ('str', 'float', 'datetime', 'card_no', 'tag', 'none', 'source')
```

3. 考虑在数据库层面添加CHECK约束
