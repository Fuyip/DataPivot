## ⚠️ 重要限制

### 核心限制说明

**数据库只有两个值**: `admin` 和 `user`

**无法区分 read 和 write**:
- 前端选择"只读" → 数据库存储 `user` → 显示为"读写"
- 前端选择"读写" → 数据库存储 `user` → 显示为"读写"
- 前端选择"管理员" → 数据库存储 `admin` → 显示为"管理员"

**实际权限效果**:
- ✅ 管理员权限: 完全控制
- ✅ 读写权限: 可以读写数据
- ❌ 只读权限: **实际上也是读写**（无法实现真正的只读）

## 📞 问题排查

### 问题1: API 返回 "Unknown column 'permission_level'"

**原因**: 代码未正确使用字段映射

**解决**: 检查 `backend/models/case.py` 中是否使用 `Column("role", ...)`

**解决**: 执行数据库迁移或修改前端UI为两级权限
