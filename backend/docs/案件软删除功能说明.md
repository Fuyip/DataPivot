# 案件软删除功能说明

## 功能概述

案件删除功能已升级为软删除机制，提供更安全的删除操作和数据恢复能力。

## 主要特性

### 1. 软删除（默认删除方式）
- **API**: `DELETE /api/v1/cases/{case_id}?confirm=true`
- **权限**: 仅系统管理员
- **特点**:
  - 需要确认参数 `confirm=true`
  - 不会真正删除数据库记录
  - 不会删除案件数据库
  - 可以恢复
  - 已删除的案件不会在正常列表中显示

### 2. 查看已删除案件
- **API**: `GET /api/v1/cases/deleted/list`
- **权限**: 仅系统管理员
- **参数**:
  - `page`: 页码（默认1）
  - `page_size`: 每页数量（默认10）
  - `search`: 搜索关键词（可选）

### 3. 恢复已删除案件
- **API**: `POST /api/v1/cases/{case_id}/restore`
- **权限**: 仅系统管理员
- **特点**:
  - 恢复后案件状态保持删除前的状态
  - 权限关系保持不变
  - 案件数据库完整保留

### 4. 永久删除（硬删除）
- **API**: `DELETE /api/v1/cases/{case_id}/permanent?confirm=true`
- **权限**: 仅系统管理员
- **特点**:
  - 需要确认参数 `confirm=true`
  - 只能删除已软删除的案件
  - 删除案件记录和所有权限
  - 删除案件数据库
  - **不可恢复**

## 数据库变更

### 新增字段
```sql
- is_deleted: BOOLEAN - 是否已删除（软删除标记）
- deleted_at: DATETIME - 删除时间
- deleted_by: INT - 删除人ID（外键关联users表）
```

### 迁移脚本
执行 `backend/migrations/add_soft_delete_to_cases.sql` 来添加新字段。

## 使用流程

### 删除案件的推荐流程
1. 首次删除：执行软删除
   ```
   DELETE /api/v1/cases/123?confirm=true
   ```

2. 如需恢复：执行恢复操作
   ```
   POST /api/v1/cases/123/restore
   ```

3. 确认不需要后：执行永久删除
   ```
   DELETE /api/v1/cases/123/permanent?confirm=true
   ```

## API 示例

### 软删除案件
```bash
curl -X DELETE "http://localhost:8000/api/v1/cases/123?confirm=true" \
  -H "Authorization: Bearer {token}"
```

### 查看已删除案件
```bash
curl -X GET "http://localhost:8000/api/v1/cases/deleted/list?page=1&page_size=10" \
  -H "Authorization: Bearer {token}"
```

### 恢复案件
```bash
curl -X POST "http://localhost:8000/api/v1/cases/123/restore" \
  -H "Authorization: Bearer {token}"
```

### 永久删除案件
```bash
curl -X DELETE "http://localhost:8000/api/v1/cases/123/permanent?confirm=true" \
  -H "Authorization: Bearer {token}"
```

## 注意事项

1. **确认参数必需**: 删除操作（软删除和永久删除）都需要 `confirm=true` 参数，防止误操作
2. **权限要求**: 所有删除和恢复操作都需要系统管理员权限
3. **永久删除限制**: 只能永久删除已软删除的案件，确保两步确认
4. **数据安全**: 软删除不会删除案件数据库，数据完整保留
5. **查询过滤**: 所有正常的案件查询API都会自动过滤已删除的案件

## 前端集成建议

1. 删除按钮应显示二次确认对话框
2. 添加"回收站"功能查看已删除案件
3. 在回收站中提供"恢复"和"永久删除"按钮
4. 永久删除前应有明确的警告提示
