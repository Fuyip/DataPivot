# 导入规则管理功能修复完成

## 修复内容

修复了前端代码中响应拦截器处理不一致的问题。

### 问题原因

`frontend/src/api/request.js` 中的响应拦截器会自动解包响应数据：
```javascript
// 拦截器返回 data，而不是完整的 {code, message, data}
return data
```

但前端代码中多处仍然使用了 `res.code === 200` 的判断方式，导致数据无法正确显示。

### 修复的文件

1. **frontend/src/views/System/ImportRuleManagement/index.vue**
   - `loadTemplates()` - 加载模板列表
   - `handleSetDefault()` - 设为默认模板
   - `handleDuplicate()` - 复制模板
   - `handleValidate()` - 验证模板
   - `handleDelete()` - 删除模板

2. **frontend/src/views/System/ImportRuleManagement/TemplateDialog.vue**
   - `handleSubmit()` - 提交模板表单

3. **frontend/src/views/System/ImportRuleManagement/FieldMappingDialog.vue**
   - `loadDataTypes()` - 加载数据类型列表
   - `loadFieldTypes()` - 加载字段类型列表
   - `loadMappings()` - 加载字段映射

4. **frontend/src/views/System/ImportRuleManagement/CleaningRuleDialog.vue**
   - `loadRules()` - 加载清洗规则
   - `handleDelete()` - 删除规则
   - `handleSubmit()` - 提交规则

5. **frontend/src/views/System/BankStatement/index.vue**
   - `fetchTemplateList()` - 获取模板列表

## 修复方式

### 修复前
```javascript
const res = await importRuleApi.getTemplateList(params)
if (res.code === 200) {
  templateList.value = res.data.items
  pagination.total = res.data.total
}
```

### 修复后
```javascript
const data = await importRuleApi.getTemplateList(params)
templateList.value = data.items || []
pagination.total = data.total || 0
```
