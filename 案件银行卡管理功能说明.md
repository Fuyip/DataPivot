# 案件银行卡管理功能实现说明

## 功能概述

在银行流水上传页面添加了对 `case_card` 表的完整管理功能，包括：
- 增删改查（CRUD）
- 数据导出（Excel）
- 模板下载
- 批量导入（Excel）

## 实现内容

### 1. 后端实现

#### 1.1 数据模型 (backend/schemas/case_card.py)
- `CaseCardBase`: 基础模型
- `CaseCardCreate`: 创建模型
- `CaseCardUpdate`: 更新模型
- `CaseCardResponse`: 响应模型
- `CaseCardListResponse`: 列表响应模型

#### 1.2 服务层 (backend/services/case_card_service.py)
- `get_case_cards()`: 获取银行卡列表（支持分页和筛选）
- `get_case_card()`: 获取单个银行卡
- `create_case_card()`: 创建银行卡
- `update_case_card()`: 更新银行卡
- `delete_case_card()`: 删除银行卡
- `export_case_cards()`: 导出为Excel
- `import_from_template()`: 从Excel模板导入
- `get_template()`: 获取导入模板

#### 1.3 API接口 (backend/api/v1/case_cards.py)
路由前缀: `/api/v1/cases/{case_id}/case-cards`

- `GET /`: 获取银行卡列表
- `GET /{card_id}`: 获取单个银行卡
- `POST /`: 创建银行卡
- `PUT /{card_id}`: 更新银行卡
- `DELETE /{card_id}`: 删除银行卡
- `GET /export/excel`: 导出Excel
- `GET /template/download`: 下载模板
- `POST /import`: 导入Excel

所有接口都包含权限检查：
- 读取操作需要 `read` 权限
- 写入操作需要 `write` 权限

### 2. 前端实现

#### 2.1 API封装 (frontend/src/api/caseCard.js)
封装了所有后端接口的调用方法

#### 2.2 管理组件 (frontend/src/components/CaseCard/CaseCardManager.vue)
功能特性：
- 数据表格展示（支持分页）
- 搜索筛选（卡号、银行名称、卡类型）
- 添加/编辑银行卡（表单验证）
- 删除银行卡（二次确认）
- 导出Excel
- 下载导入模板
- 批量导入Excel

#### 2.3 页面集成 (frontend/src/views/System/BankStatement/index.vue)
在银行流水上传页面顶部添加了可折叠的案件银行卡管理区域：
- 默认收起状态
- 点击展开/收起按钮切换显示
- 自动关联当前选择的案件

### 3. 数据库表结构

`case_card` 表字段：
- `id`: 主键
- `case_no`: 案件编号
- `card_no`: 卡号（唯一索引）
- `bank_name`: 银行名称
- `card_type`: 卡类型
- `source`: 来源信息
- `user_id`: 用户ID
- `batch`: 批次
- `is_in_bg`: 是否在后台
- `is_main`: 是否为主卡
- `add_date`: 添加日期

## 使用说明

### 1. 添加单个银行卡
1. 在银行流水上传页面展开"案件银行卡管理"
2. 点击"添加银行卡"按钮
3. 填写表单信息（卡号为必填项）
4. 点击确定保存

### 2. 批量导入银行卡
1. 点击"下载模板"按钮获取Excel模板
2. 按照模板格式填写数据（卡号为必填列）
3. 点击"导入"按钮上传填好的Excel文件
4. 系统会显示导入结果（成功数、失败数、错误信息）

### 3. 导出银行卡数据
1. 点击"导出"按钮
2. 系统会下载包含所有银行卡数据的Excel文件

### 4. 编辑和删除
- 点击表格中的"编辑"按钮修改银行卡信息
- 点击"删除"按钮删除银行卡（需要二次确认）

## 权限控制

- 查看银行卡列表：需要案件的 `read` 权限
- 添加/编辑/删除银行卡：需要案件的 `write` 权限
- 导入/导出：读取需要 `read` 权限，导入需要 `write` 权限
- 超级管理员和管理员拥有所有权限

## 注意事项

1. 卡号具有唯一性约束，重复导入会失败
2. 导入时只显示前10条错误信息
3. 导出的Excel文件包含所有字段
4. 模板中的示例数据可以参考填写格式
5. 批次、是否在后台、是否主卡字段为数字类型（0或1）

## 技术栈

- 后端：FastAPI + SQLAlchemy + Pandas + OpenPyXL
- 前端：Vue 3 + Element Plus
- 数据库：MySQL
