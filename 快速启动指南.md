## 安装必需服务

### 1. 安装Redis
```bash
# 使用Homebrew安装
brew install redis

# 启动Redis服务
brew services start redis

# 验证Redis是否运行
redis-cli ping  # 应返回 PONG
```

### 2. 检查MySQL
```bash
# 检查MySQL是否已安装
mysql --version

# 如果未安装，使用Homebrew安装
brew install mysql

# 启动MySQL服务
brew services start mysql
```

## 启动步骤

### 步骤1：执行数据库迁移
```bash
cd /Users/yipf/DataPivot项目/DataPivot

# 执行迁移脚本
mysql -u root -p datapivot < backend/migrations/add_bank_statement_tasks.sql

# 验证表是否创建成功
mysql -u root -p -e "USE datapivot; SHOW TABLES LIKE 'bank_statement_tasks';"
```

### 步骤2：启动Celery Worker
**打开新的终端窗口**，执行：
```bash
cd /Users/yipf/DataPivot项目/DataPivot
source venv/bin/activate
celery -A backend.core.celery_app worker --loglevel=info
```

保持这个终端窗口运行。

### 步骤3：启动FastAPI应用
**再打开一个新的终端窗口**，执行：
```bash
cd /Users/yipf/DataPivot项目/DataPivot/backend
source ../venv/bin/activate
python main.py
```

或使用uvicorn：
```bash
cd /Users/yipf/DataPivot项目/DataPivot
source venv/bin/activate
uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000
```

### 步骤4：访问API文档
打开浏览器访问：
- Swagger UI: http://localhost:8000/docs
- ReDoc: http://localhost:8000/redoc

## 测试接口

### 1. 登录获取Token
```bash
curl -X POST "http://localhost:8000/api/v1/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"username": "your_username", "password": "your_password"}'
```

### 2. 上传银行流水文件
```bash
# 替换 {case_id} 和 {token}
curl -X POST "http://localhost:8000/api/v1/cases/{case_id}/bank-statements/upload" \
  -H "Authorization: Bearer {token}" \
  -F "files=@/path/to/your/file.zip"
```

### 3. 查询任务进度
```bash
# 替换 {case_id}、{task_id} 和 {token}
curl -X GET "http://localhost:8000/api/v1/cases/{case_id}/bank-statements/tasks/{task_id}" \
  -H "Authorization: Bearer {token}"
```

## 故障排查

### Redis连接失败
```
错误: Error connecting to Redis
解决:
1. 检查Redis是否运行: redis-cli ping
2. 检查Redis配置: config.py中的REDIS_HOST和REDIS_PORT
```

### Celery任务不执行
```
问题: 任务状态一直是pending
解决:
1. 检查Celery Worker是否运行
2. 查看Celery Worker终端的日志输出
3. 确认Redis连接正常
```

### 数据库连接失败
```
问题: Can't connect to MySQL server
解决:
1. 检查MySQL是否运行: brew services list | grep mysql
2. 检查.env文件中的数据库配置
3. 测试数据库连接: mysql -u root -p
```

### 文件上传失败
```
问题: 413 Request Entity Too Large
解决: 增加FastAPI的文件大小限制（默认已支持大文件）
```
