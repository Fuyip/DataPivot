# 案件数据库表缺失问题修复完成报告

## 问题描述

用户报告：新建案件缺失 `bank_all_statements` 主表和 `bank_all_statements_tmp` 临时表。

## 问题根源

经过深入调查，发现问题的根本原因是 **SQL 模板文件中的语法错误**：

在 `sql/schema/case_template.sql` 文件中，三个表的 `rival_trade_balance` 字段被错误地定义为：

```sql
`rival_trade_balance` double CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci DEFAULT NULL COMMENT '对手交易余额',
```

**问题**：`double` 数值类型不能使用 `CHARACTER SET` 和 `COLLATE` 属性，这些属性只适用于字符串类型。

这导致 MySQL 在执行 CREATE TABLE 语句时报错：
```
(1064, "You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near 'CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci DEFAULT NULL COMMENT '对手交' at line 24")
```

## 受影响的表

以下三个表因为这个语法错误无法创建：

1. `bank_all_statements` - 银行流水主表
2. `bank_all_statements_tmp` - 银行流水临时表
3. `bank_all_statements_with_info` - 带信息的银行流水表

## 修复方案

### 1. 修复 SQL 模板文件

**文件**：`sql/schema/case_template.sql`

**修改**：将所有 `rival_trade_balance` 字段的定义从：
```sql
`rival_trade_balance` double CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci DEFAULT NULL COMMENT '对手交易余额',
```

修改为：
```sql
`rival_trade_balance` double DEFAULT NULL COMMENT '对手交易余额',
```

**影响范围**：共修改了 3 处（对应三个缺失的表）
