### 数据库状态：❓ 未确认

**关键问题**：需要确认当前数据库实际使用的字段名

有两种可能：

#### 情况1：数据库使用 `permission_level` 字段
- SQL 初始化脚本定义的是 `permission_level`
- 如果数据库是用初始化脚本创建的，字段名应该是 `permission_level`
- **结论**：代码和数据库一致，可以直接使用 ✅

#### 情况2：数据库使用 `role` 字段
- 如果数据库是早期版本创建的，可能使用 `role` 字段
- **结论**：需要执行数据库迁移或回退代码 ⚠️

## 如何确认数据库字段名

### 方法1：运行检查脚本（需要 MySQL 服务）

```bash
backend/venv/bin/python sql/migrations/check_database.py
```

这个脚本会：
- 检查表结构中的字段名
- 显示现有权限数据分布
- 明确告诉你使用的是哪个字段

### 方法2：查看后端日志

启动后端服务，如果看到类似错误：
```
sqlalchemy.exc.OperationalError: (pymysql.err.OperationalError) (1054, "Unknown column 'permission_level' in 'field list'")
```
说明数据库使用的是 `role` 字段。

### 方法3：直接查询数据库

如果可以连接 MySQL：
```sql
DESCRIBE datapivot.user_case_permissions;
```
