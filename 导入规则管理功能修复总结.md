# 导入规则管理功能问题修复总结

## 修复的问题

### 1. 字段映射加载失败 ✅

**问题描述**：前端显示"加载字段映射失败"，后端返回500错误。

**根本原因**：数据库中有4条记录的 `field_type` 字段值为 `'n'`，不符合Pydantic Schema验证规则。

**修复方案**：
```sql
UPDATE import_field_mappings
SET field_type = 'none'
WHERE field_type NOT IN ('str', 'float', 'datetime', 'card_no', 'tag', 'none', 'source')
```

**修复结果**：字段映射API现在可以正常返回数据。

---

### 2. 数据库字段名无法编辑 ✅

**问题描述**：在字段映射配置页面，数据库字段名列是只读的，新增字段时无法输入字段名。

**根本原因**：`FieldMappingDialog.vue` 中数据库字段名列使用了 `prop="db_field_name"`，没有提供输入框。

**修复方案**：将数据库字段名列改为可编辑：
```vue
<!-- 修改前 -->
<el-table-column prop="db_field_name" label="数据库字段" width="180" />

<!-- 修改后 -->
<el-table-column label="数据库字段" width="180">
  <template #default="{ row }">
    <el-input v-model="row.db_field_name" size="small" placeholder="字段名" />
  </template>
</el-table-column>
```

**修复结果**：现在可以在表格中直接编辑数据库字段名。

---

### 3. API响应拦截器处理不一致 ✅

**问题描述**：多个页面的API调用使用了 `res.code === 200` 判断，但响应拦截器已经解包数据。

**根本原因**：`request.js` 中的响应拦截器返回 `data` 而不是完整响应对象。

**修复的文件**：
1. `ImportRuleManagement/index.vue` - 5个函数
2. `ImportRuleManagement/TemplateDialog.vue` - 1个函数
3. `ImportRuleManagement/FieldMappingDialog.vue` - 3个函数
4. `ImportRuleManagement/CleaningRuleDialog.vue` - 3个函数
5. `BankStatement/index.vue` - 1个函数

**修复示例**：
```javascript
// 修改前
const res = await importRuleApi.getTemplateList(params)
if (res.code === 200) {
  templateList.value = res.data.items
}

// 修改后
const data = await importRuleApi.getTemplateList(params)
templateList.value = data.items || []
```

## 使用说明

### 1. 创建新模板

1. 访问"系统管理" → "导入规则管理"
2. 点击"新增模板"按钮
3. 填写模板名称和描述
4. 保存后会自动刷新列表

### 2. 配置字段映射

1. 在模板列表中点击"字段映射"按钮
2. 选择数据类型标签页（人员信息、账户信息、交易明细等）
3. 点击"添加字段"按钮
4. 填写：
   - **数据库字段名**：目标数据库表的字段名（现在可编辑）
   - **CSV列名**：源文件中的列名
   - **字段类型**：选择数据处理方式
   - **必填**：是否必须有值
   - **默认值**：可选的默认值
5. 可以拖拽行来调整顺序
6. 点击"保存配置"

### 3. 配置清洗规则

1. 在模板列表中点击"清洗规则"按钮
2. 点击"添加规则"
3. 填写：
   - **规则名称**：规则的描述性名称
   - **规则类型**：通用清洗 或 日期时间清洗
   - **正则表达式**：用于清洗数据的正则表达式
   - **规则说明**：可选的详细说明
4. 可以使用"测试工具"验证正则表达式
5. 保存规则

### 4. 使用模板导入数据

1. 访问"银行流水上传"页面
2. 选择案件
3. 点击"上传流水文件"
4. 在"导入规则"下拉框中选择模板
5. 上传文件

## 技术细节

### 支持的字段类型

| 值 | 标签 | 说明 |
|---|---|---|
| `str` | 字符串 | 保持原样的文本字段 |
| `float` | 浮点数 | 数值字段，空值转为NULL |
| `datetime` | 日期时间 | 日期时间字段，清理非数字字符 |
| `card_no` | 卡号 | 银行卡号，提取第一个数字串 |
| `tag` | 收付标志 | 转换为in/out标识 |
| `none` | 不处理 | 跳过该字段 |
| `source` | 来源标记 | 特殊标记字段 |

### 支持的数据类型

1. 人员信息 - 银行账户持有人的个人信息
2. 账户信息 - 银行账户的基本信息
3. 子账户信息 - 银行子账户信息
4. 强制措施信息 - 账户冻结等强制措施信息
5. 交易明细 - 银行流水交易明细
6. 失败信息 - 查询失败的任务信息
