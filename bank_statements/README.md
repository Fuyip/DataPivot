# 银行流水数据处理目录

## 目录结构

```
bank_statements/
├── raw/              # 原始压缩包存放目录（待处理）
├── processing/       # 正在处理的文件目录
├── processed/        # 已处理完成的文件归档目录
├── error_files/      # 处理失败的文件目录
└── logs/            # 日志文件目录
```

## 使用说明

1. **raw/** - 将银行返回的压缩包文件放入此目录
2. **processing/** - 程序自动解压和处理文件的工作目录
3. **processed/** - 处理成功后，文件会移动到此目录归档
4. **error_files/** - 处理失败的文件会自动复制到此目录
5. **logs/** - 程序运行日志存放目录

## 工作流程

1. 上传压缩包到 `raw/` 目录
2. 运行清洗脚本，程序会自动解压并处理
3. 处理成功的文件移至 `processed/`
4. 处理失败的文件复制到 `error_files/`
5. 查看 `logs/` 目录中的日志了解处理详情
